ACM FUN — React.js Project

ACM FUN is an initiative by PECACM for beginners to build interactive web tools and games using React.js.

Live Demo

Description

ACM FUN engages beginners with interactive Activities and Games built in React. Activities include simple web tools and generators. Games include Wordle, Tic-Tac-Toe, and more.

Features

  • Interactive React UI
  • Beginner-friendly contribution flow
  • Two sections: Activities and Games
  • Community collaboration and learning

Tech Used

React.js for the frontend. Vercel for deployment.

How to Start

  1. Clone the repository: 
    git clone https://github.com/PEC-CSS/acm-fun.git
cd acm-fun
  2. Install dependencies: 
    npm install
  3. Run the app: 
    npm start
  4. Open http://localhost:3000 in your browser.

Where to Add a New Game or Activity

Follow the directory structure and add components as React pages:

src/pages/games/      // add game components here
src/pages/activities/ // add activity components here
src/styles/pages/games/       // css for games
src/styles/pages/activities/  // css for activities
src/data/content.js           // add an entry so the component appears on the site

Contribution Steps

  1. Fork the repository and create a branch: 
    git checkout -b feature/your-feature-name
  2. Create your React component in the appropriate folder and its CSS file.
  3. Add an entry in src/data/content.js with required metadata so the new page is picked up by the site.
  4. Run locally and verify UI, responsiveness, and accessibility.
  5. Commit & push your changes and open a Pull Request with a clear description.

content.js — Add Entry Example

{
  id: "my-awesome-game",
  title: "My Awesome Game",
  type: "game",
  componentPath: "games/MyAwesomeGame",
  thumbnail: "/images/my-awesome-game.png",
  tags: ["puzzle","react"],
  difficulty: "beginner"
}

Best Practices

  • Keep components modular and small.
  • Follow existing coding style and naming conventions.
  • Make CSS scoped to the page and reuse global variables when available.
  • Test on mobile and desktop; ensure keyboard accessibility and proper focus management.
  • Document how to run and test the component in a short README or the PR description.

Do Not

  • Do not add large media assets without optimization.
  • Do not include secrets or API keys in the repo.
  • Do not change unrelated files; keep PRs minimal and focused.
